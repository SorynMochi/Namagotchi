## Recommended first structural consolidation target

Start with **Home / Nami care**. It has substantial duplication (`.care-card`, `.nami-card`, `.nami-care-summary`, `.care-status-compact`, `.nami-message-log-card`, and care buttons), is visually isolated to the Home screen, has clear route/state checks (`is-sleep-stage`, `is-idle-stage`, care summary, log, action queue), and can likely be changed inside a rollback-bounded commit under roughly 500 changed lines. Chat and Wardrobe have larger payoff, but both depend on more interaction states, dynamic DOM, modal behavior, and theme overrides. Top rail has high duplication but is entangled with breakpoint migrations and `!important` repair rules.
